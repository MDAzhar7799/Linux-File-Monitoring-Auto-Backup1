Linux File Monitoring & Auto-Backup System

GuardianFS is a powerful, open-source solution for real-time file monitoring, automatic backups, and system integrity protection on Linux servers.

Live Demo Get Started

Powerful Features

GuardianFS provides a comprehensive set of tools to monitor, protect, and backup your critical system files and directories.

Real-time Monitoring

Monitor files and directories for changes in real-time using inotify, with instant notifications for modifications, deletions, or access attempts.

Automatic Backups

Schedule automated backups with configurable retention policies. Supports incremental backups to save storage space.

Smart Notifications

Receive alerts via email, Slack, or webhook when critical changes are detected or backup operations complete.

Dashboard & Analytics

Interactive web dashboard to visualize file changes, backup status, and system health metrics.

Version Control Integration

Automatically commit changes to Git repositories with detailed change logs for easy rollback and audit trails.

Security & Integrity

File integrity checking with hash verification and automated restoration from backups when tampering is detected.

Quick Setup Guide

Get GuardianFS running on your Linux system in just a few simple steps.

1

Download GuardianFS

Save the following Python script as guardianfs.py on your system:

# Python script will be displayed here
2

Run the Script

Execute the script from your terminal:

python3 guardianfs.py

Or run it as a background service:

nohup python3 guardianfs.py > guardianfs.log 2>&1 &
3

Configure Monitoring

Edit the configuration section in the script to specify directories to monitor:

# Configuration MONITOR_DIRS = ["/home/user/documents", "/var/www"] BACKUP_DIR = "/backups" CHECK_INTERVAL = 5 # seconds
4

Start Monitoring

The script will automatically start monitoring and create backups when changes are detected.

# Check if monitoring is active ps aux | grep guardianfs.py # View logs tail -f guardianfs.log
